## STEP 1: 研究室の Apple Developer アカウントでの自分の役割(Role)を確認(所要 5分)

研究室(組織)アカウントに招待済みなので **個人での $99 加入は不要**です。ただし EAS がビルド時に
署名証明書・プロビジョニングプロファイルを自動作成するため、自分の役割に**証明書を作る権限**が必要です。

1. https://appstoreconnect.apple.com にサインイン →「ユーザとアクセス」
2. 自分の名前を開いて **役割(Role)** を確認

| Role | 開発ビルド/ビルド署名 | メモ |
|---|---|---|
| Admin / Account Holder | ◎ 問題なし | そのまま全 STEP を進められる |
| App Manager | ○ ほぼ可 | 証明書作成が制限される場合のみ管理者に依頼 |
| Developer | △ 制限あり | 端末登録・証明書作成で管理者依頼が必要になることがある |

3. あわせて「証明書、識別子、プロファイルへのアクセス」が有効になっているか確認(なければ管理者に依頼)

✅ 完了確認: 自分が **App Manager 以上**であればこの先の手順は基本的に自分で完結できる

## STEP 4: 開発ビルドを作って実機で動作確認(所要 30〜60分、大半はビルド待ち)

**ここが Expo Go の置き換え。** 一度この開発ビルドを iPhone に入れれば、以降のコード修正は
Metro サーバ経由で即反映されます(Expo Go と同じ感覚で開発できる)。

### 4-1. iPhone を EAS に端末登録する(初回のみ)

開発ビルドを実機に入れるには、その iPhone を Apple Developer アカウントに登録する必要があります。

```bash
eas device:create
```

1. `Apple Developer` へのログインを求められる → **研究室アカウントの Apple ID** でログイン
   (2ファクタ認証のコードは iPhone に届く)
2. 登録用の **URL / QR コード**が表示される → **iPhone のカメラ or Safari で開く**
3. 画面の指示に従い **構成プロファイルをインストール**(設定 → プロファイルがダウンロードされました → インストール)
   → これで iPhone の UDID がアカウントに登録される

✅ 完了確認: Apple Developer の「デバイス」一覧に自分の iPhone が出る

### 4-2. 開発ビルドをクラウドで作成

```bash
eas build --profile development --platform ios
```

初回のみ、途中で以下を聞かれます(Claude Code が実行するので、あなたは認証入力だけ):

| プロンプト | 答え方 |
|---|---|
| Apple アカウントにログインしますか | **Y** → 研究室アカウントの Apple ID + パスワード |
| 2ファクタ認証(6桁) | **iPhone に届くコード**を入力 |
| どのデバイスを含めるか | 4-1 で登録した iPhone を選択(スペースキーで選択 → Enter) |
| Distribution Certificate を生成しますか | **Y**(EAS が自動生成) |
| Provisioning Profile を生成しますか | **Y** |

→ その後クラウドでビルドが走る(15〜30分)。進捗 URL がターミナルに表示される。

### 4-3. iPhone にインストール & デベロッパモード有効化

1. ビルド完了時にターミナル/ブラウザに表示される **QR コード**を iPhone のカメラで読む
2. 「インストール」→ ホーム画面に「筋トレログ」の開発ビルドが入る
3. **デベロッパモードを有効化する**(iOS 16 以降で開発ビルドの起動に必須):
   - アプリを起動すると「**デベロッパモードが必要です**」と出る → 「OK」
   - **設定 → プライバシーとセキュリティ → 一番下の「デベロッパモード」** をタップ
   - トグルを **ON** → 「再起動」→ iPhone が再起動
   - 再起動後ロック解除すると「デベロッパモードをオンにしますか?」→ **「オンにする」** → パスコード入力
   - ※「デベロッパモード」項目が見当たらないときは、一度 iPhone を再起動してから再確認
4. (出た場合のみ)起動時に「**信頼されていないデベロッパ**」と出たら:
   - **設定 → 一般 → VPN とデバイス管理** → 「デベロッパ App」の下の証明書(`Apple Development: ...` / `manamu Inc`)をタップ → 「"◯◯"を信頼」→「信頼」
   - ※端末登録済みの ad-hoc 配布では出ないことも多い。出なければ不要

### 4-4. 開発サーバに繋いで動作確認

> 前提: Mac と iPhone が**同じ Wi-Fi**(大学 Wi-Fi は端末間通信が遮断されがち。自宅 Wi-Fi かテザリング推奨。
> ダメなら Claude Code に「トンネルモードで起動して」と頼む → `npx expo start --dev-client --tunnel`)

1. Mac で開発サーバを起動(Claude Code に「開発サーバを起動して」でOK):
   ```bash
   npx expo start --dev-client
   ```
2. iPhone で「筋トレログ」(開発ビルド)を起動 → 自動で Metro に繋がる
   (繋がらなければアプリ内に出る一覧から Mac のサーバを選ぶ / QR を読む)

### 動作確認チェックリスト

- [ ] 記録タブ: 種目チップを選び、重量 60・回数 10 を入れて「セットを記録」→ 下のリストに追加される
- [ ] 同じ種目でもう1セット記録できる
- [ ] 記録の「削除」ボタン → 確認ダイアログ → 消える
- [ ] 履歴タブ: 今日の日付でグループ表示される
- [ ] **【課題の肝・永続化テスト】** アプリをアプリスイッチャーから完全終了 → 開き直す → **記録が残っている**
- [ ] 機内モードにしても記録の閲覧・追加ができる(通信不要 = 端末内保存の証明)

🐛 不具合を見つけたら Claude Code に伝える → 修正 → 保存すると自動リロードされるので再確認
(コードを変えただけなら再ビルド不要。ネイティブ設定を変えたときだけ 4-2 を再実行)

## STEP 5: TestFlight へのビルド&提出(所要 30分〜1時間、大半はビルド待ち)

動作確認が済んだら、最終配布用のビルドを作って App Store Connect に上げます。
**コマンドは Claude Code が実行します。** あなたは途中の認証入力と確認プロンプトへの回答だけ。
Claude Code に「TestFlight へのビルドと提出を進めて」と言えば開始します。

### 5-1. 本番ビルド(`eas build --platform ios --profile production`)

STEP 4 で証明書は作成済みなので、多くの場合そのまま流れます。聞かれたら:

| プロンプト | 答え方 |
|---|---|
| Apple ID ログイン | 研究室アカウント(セッションが残っていれば聞かれない) |
| Provisioning Profile 生成 | **Y** |

→ クラウドビルド(15〜30分待ち)

### 5-2. App Store Connect へ提出(`eas submit --platform ios --latest`)

| プロンプト | 答え方 |
|---|---|
| Apple ID での認証 | 5-1 と同様(セッションが残っていれば聞かれない) |
| App Store Connect にアプリレコードを作成しますか | **Y**(アプリ名は app.json から自動設定) |

✅ 完了確認: 「Submitted!」と表示され、しばらくすると Apple から processing 完了メールが届く

## STEP 6: TestFlight でテスターに権限付与(所要 15分 + 反映待ち)

**ここが課題の「権限付与(招待)」の実体。** すべて App Store Connect のブラウザ UI 操作。

### 6-1. ビルドの処理完了を待つ

1. https://appstoreconnect.apple.com →「マイアプリ」→「筋トレログ」→ 上部の **TestFlight** タブ
2. アップロードしたビルドが「処理中」→ 10〜30分で「テスト可能」等に変わる
   - 「輸出コンプライアンス情報が必要」と出たら → app.json で暗号化申告済みなので通常は自動解決。
     聞かれたら「暗号化を使用していない」を選択

### 6-2. テスターをユーザとして追加(=権限付与)

1. App Store Connect トップ → **「ユーザとアクセス」**
2. 「+」→ テスターにしたい研究室メンバーの氏名・メールアドレスを入力
3. 役割は最小限でOK(TestFlight で配るだけなら強い権限は不要)
4. 「招待」→ 相手に招待メールが届く → 相手が承諾

> 💡 「内部テスター」にできるのは App Store Connect にユーザ登録された人(最大100人)。
> ユーザ登録せずに配りたい場合は「外部テスト」(6-4)を使う。

### 6-3. 内部テストグループを作ってビルドを配る

1. 「筋トレログ」→ TestFlight タブ →「内部テスト」の **「+」**
2. グループ名(例: `研究室メンバー`)を作成
3. 「テスター」→「+」→ 6-2 で追加したユーザを選択
4. 「ビルド」→「+」→ STEP 5 のビルドを選択
5. テスターに **TestFlight の招待メール**が自動送信される(内部テストは**審査不要**で即配信)

### 6-4. (任意)外部テスターに配る場合

- TestFlight タブ →「外部テスト」→ グループ作成 → メールアドレスだけで最大10,000人招待可能
- ただし初回は **Beta App Review(審査)** が必要で 1〜2 日かかる
- 研究室内での配布なら 6-3 の内部テストで十分

## トラブルシューティング

| 症状 | 対処 |
|---|---|
| 「incompatible with this version of Expo Go」 | Expo Go は SDK 57 非対応。STEP 4 の**開発ビルド**を使う(このアプリを Expo Go で開くことはできない) |
| 開発ビルドが Metro に繋がらない | Mac と iPhone が同じ Wi-Fi か確認。ダメなら `npx expo start --dev-client --tunnel` を Claude Code に依頼 |
| `eas device:create` で iPhone が登録できない | Safari で登録 URL を開く / 構成プロファイルのインストールを「設定」アプリで完了させる |
| `eas build` で Apple ID ログインに失敗 | developer.apple.com に一度ブラウザでログインして規約更新に同意してから再実行 |
| 証明書/プロファイルが作れない(権限エラー) | 組織アカウントでの自分の Role が Developer だと制限される。管理者に App Manager 以上へ変更を依頼 |
| ビルドが Queue のまま進まない | EAS 無料枠の混雑。待つ(混雑時30分以上) |
| TestFlight にビルドが出てこない | アップロード後の処理に最大1時間かかることがある。Apple のメールを待つ |
| 「Missing Compliance」と表示される | app.json の `usesNonExemptEncryption: false` 設定済み。出たらビルドの「管理」→「暗号化を使用していない」を選択 |
| TestFlight/開発ビルドが起動しなくなった | ビルドの有効期限切れ(TestFlight 90日 / 開発ビルドは登録端末の証明書有効期間)。再ビルドが必要 |
