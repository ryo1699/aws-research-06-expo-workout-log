## A. 記録機能の強化(筋トレMEMO風・適切な分量)

「筋トレMEMO」の定番機能から、シンプルな端末内アプリに馴染むものを厳選して入れる。

| # | 機能 | 内容 | 備考 |
|---|---|---|---|
| A-1 | **部位カテゴリ + 種目のカスタム管理** | 種目を「胸/背中/脚/肩/腕/体幹」等の部位で整理。ユーザーが種目を追加/削除できる | 種目マスタを DB 化(`exercises` テーブル)。既存の固定リストは初期データとして投入 |
| A-2 | **推定1RM自動計算** | 記録した重量×回数から Epley 式で推定1RMを算出・表示 | `1RM = w × (1 + reps/30)`。自重(0kg)は対象外 |
| A-3 | **前回記録の表示 & コピー** | 種目を選ぶと「前回: 60kg×10」を表示。タップで入力欄に流し込み | 同じ種目の最新セットを引くだけ |
| A-4 | **種目別の推移グラフ** | 種目ごとに推定1RM(or 総ボリューム)の推移を折れ線/棒で表示 | **View ベースの軽量自作グラフ**でネイティブ依存を避ける |
| A-5 | **セット間インターバルタイマー** | 「セット記録」後にカウントダウン開始。0で通知 | 端末内タイマー。通知は B の仕組みを流用 |
| A-6 | **メモ欄(任意)** | 1セットに短いメモを添付できる | `sets.memo` カラム追加 |

**スコープ外(今回やらない)**: SNS共有・有酸素運動・kg/lbs切替・Apple Watch連携(外部依存/肥大化のため)。

## B. アバター育成 + 通知(ゲーミフィケーション)

「スマホをやめたら魚が育つ」的に、**筋トレを継続するほどアバターが成長・進化**する。

### B-1. 成長ロジック

- 成長の駆動源 = **累計トレーニング日数**(その日 1 セット以上記録した日数) + **連続記録(streak)**。
  - 「継続するほど育つ」= 日数ベースが Forest 的で分かりやすい(1日に大量記録しても飛び級しない)。
- 進化はしきい値で段階的に起こる。進化した瞬間に「進化演出 + 通知」。

### B-2. アバター 3系統 × 成長段階(絵文字ベース)

要望の「鳥の雛 → フェニックス → ドラゴン」を3系統として実装:

| 段階 | 累計日数(目安) | 絵文字 | 呼称 |
|---|---|---|---|
| 卵 | 0 | 🥚 | たまご |
| **鳥系** ヒナ | 1 | 🐣 | ヒナ |
| 鳥系 ひよこ | 3 | 🐥 | ひよこ |
| 鳥系 成鳥 | 7 | 🐦 | 成鳥 |
| **フェニックス系** 幼 | 14 | 🐦‍🔥 | 火の鳥(幼) |
| フェニックス系 成 | 30 | 🔥🦅 | フェニックス |
| **ドラゴン系** 幼 | 60 | 🐲 | 子竜 |
| ドラゴン系 成 | 100 | 🐉 | ドラゴン |

※ しきい値・呼称・絵文字は実装時に微調整可。段階定義は `src/avatar.ts` に一元化。

### B-3. ホーム(アバター)タブ 追加

- 現在のアバター(大きく表示) + 呼称
- 継続日数 / 連続記録(streak) / 次の進化まであと何日
- 進化直後は「進化した!」バナー

### B-4. アバターからの通知(応援機能)

- **毎日決まった時刻**(ユーザー設定)に、アバターが筋トレを促すローカル通知。
  - 例: 「今日はまだ鍛えてないよ🐦‍🔥 一緒に頑張ろう!」(その日未記録のときだけ)
- **streak が途切れそう**な時のリマインド。
- **進化した瞬間**のお祝い通知。
- 実装は `expo-notifications`(ローカル通知のみ。サーバー不要)。

> ⚠️ **`expo-notifications` はネイティブモジュール** → 追加後は **開発ビルドの再作成(`eas build --profile development`)が必要**。
> A の機能と B-1〜B-3(通知以外)は JS のみで現在の開発ビルドにホットリロードで反映できるが、
> **通知(B-4)だけは再ビルドが要る**。

## データモデルの変更(SQLite)

```sql
-- 種目マスタ(A-1)
CREATE TABLE exercises (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL UNIQUE,
  category TEXT NOT NULL,      -- 部位: 胸/背中/脚/肩/腕/体幹
  sort_order INTEGER NOT NULL,
  is_builtin INTEGER NOT NULL DEFAULT 0
);

-- sets に memo 追加(A-6)
ALTER TABLE sets ADD COLUMN memo TEXT;

-- 設定(通知時刻/ON-OFF)(B-4)
CREATE TABLE settings (
  key TEXT PRIMARY KEY,
  value TEXT NOT NULL
);
```

- **継続日数 / streak は `sets` から集計**(distinct な現地日付)で導出 → 二重管理を避ける。
- マイグレーションは `db.ts` の `migrateDbIfNeeded` に `user_version` ベースで段階追加。既存データは保持。
